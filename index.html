Día 21 de octubre: Los Frameworks

Notas de Clase:

¿Que son los Frameworks?

Un framework es una herramienta fundamental en el desarrollo de software que proporciona una estructura y componentes predefinidos para facilitar y acelerar la creación de aplicaciones. Estos actúan como un esqueleto sobre el cual los desarrolladores pueden construir sus proyectos, evitando la necesidad de escribir código repetitivo y asegurando buenas prácticas y consistencia en el código.

Los frameworks se utilizan en el ámbito de programación de aplicaciones desde hace décadas. Recientemente se han usado para programar y diseñar aplicaciones web, por lo que ya existen decenas de frameworks para CSS.

Los frameworks incluyen bibliotecas, módulos y herramientas que ayudan en tareas comunes como la manipulación de datos, la gestión de sesiones, la seguridad y la autenticación. Además, suelen estar respaldados por grandes comunidades de desarrolladores que contribuyen con recursos, soporte y actualizaciones continuas.

Frameworks populares:

  • Bootstrap:
    Es un framework de CSS y JavaScript para diseño web responsivo. Incluye una amplia gama de componentes como formularios, botones, menús de navegación entre otros elementos y además esta optimizado para dispositivos móviles y de escritorio.
  • Foundation:
    Tambien es un framework CSS y JavaScript enfocado en el diseño responsive y mobile-first. Incluye una cuadrícula flexible y componentes de interfaz de usuario y está diseñado para funcionar en una amplia variedad de dispositivos, desde móviles hasta desktops.
  • Bulma:
    Este es un framework CSS basado en Flexbox, conocido por su simplicidad y modularidad. No incluye componentes JavaScript, es ideal para crear interfaces web responsivas y atractivas.
  • Tailwind:
    Un framework de CSS de utilidad que permite crear diseños personalizados usando clases de utilidad. Facilita la creación de interfaces sin escribir CSS adicional, adecuado para cualquier dispositivo gracias a su enfoque en clases de utilidad.
  • Pure CSS:
    Es un conjunto de módulos CSS pequeños y responsivos. Diseñado para ser minimalista, fácil de personalizar y optimizado para dispositivos móviles y de escritorio.
  • Skeleton:
    Este es un framework CSS ligero para proyectos pequeños. Incluye una cuadrícula básica y estilos para elementos HTML comunes, funciona literal como un esqueleto para diseño web.
  • Materialize:
    Framework CSS basado en Material Design de Google que incluye componentes interactivos y estilos predefinidos. Diseñado para ofrecer una experiencia consistente en dispositivos móviles y de escritorio.
  • Material Design Lite:
    Es una Implementación ligera de Material Design. Proporciona componentes y estilos básicos para crear aplicaciones web.
  • UI Kit:
    Es un framework de CSS y JavaScript modular para el desarrollo rápido de interfaces web y además incluye una amplia gama de componentes y utilidades. Compatible con una variedad de dispositivos, desde móviles hasta desktops.
  • Sass:
    Preprocesador CSS que permite usar variables, anidamiento, mixins y más para escribir CSS más eficiente y mantenible. No es un framework de diseño, pero se utiliza para mejorar la escritura de CSS en cualquier proyecto web.

Ideas Clave:

  • Los framework pueden funcionar para lograr la responsividad y evitar la necesidad de @media.
  • Los frameworks proporcionan estructuras predefinidas y herramientas que aceleran el desarrollo, permitiendo a los desarrolladores enfocarse más en la codificación de html.
  • Los frameworks populares suelen tener grandes comunidades de desarrolladores que contribuyen con plugins, extensiones y soporte, facilitando la resolución de problemas y la implementación de nuevas funcionalidades.
  • Utilizar un framework bien establecido puede facilitar el mantenimiento y la escalabilidad del código, ya que sigue patrones y convenciones que son ampliamente comprendidos y documentados.

Resumen:

Un framework es un conjunto de herramientas, librerías, convenciones y buenas practicas que pretenden encapsular las tareas repetitivas en módulos genéricos fácilmente reutilizables.

Los frameworks son herramientas esenciales en el desarrollo de software, ya que proporcionan estructuras y componentes predefinidos que aceleran el proceso de creación de aplicaciones. Además de ahorrar tiempo y esfuerzo, facilita la resolución de problemas, la implementación de nuevas funcionalidades y mejora el mantenimiento y la escalabilidad del código.

Día 23 de octubre: Mobile-First

Notas de Clase:

Historia del Mobile-First:

Una investigación realizada por la GSMA (Asociacion Global de Sistemas de Comunicación Movil) afirma que el uso de internet a través de dispositivos móviles crecerá un 50% hasta 2020. En 2015, Google ya hablaba sobre esta tendencia, y priorizo los sitios web que son mobile friendly (sitios web amigables para dispositivos móviles) en los resultados de las búsquedas.

Los dispositivos móviles garantizan la practicidad que se busca, pero para que la experiencia en internet sea buena, es necesario que la navegación sea pensada para estos dispositivos. Si las personas pasan más tiempo navegando en internet por medio del teléfono móvil; lo más sensato sería crear primero un sitio web optimizado para los dispositivos móviles.

Pensando en esto, Luke Wroblewski (director de productos de Google) en 2011, desarrollo el concepto Mobile First, cuando publico el libro Mobile First. Su propuesta era crear un sitio web pensado primero para dispositivos móviles y después ajustarlo para la computadora.

Que es el Mobile First:

El modelo Mobile first es una filosofía que perfecciona el proceso de diseño de cualquier sitio web, buscando mejorar el diseño web adaptable. Esta nueva forma de diseñar, surge de los dispositivos móviles y de cualquier pantalla de tamaño menor, en la que se parte de una escala en aumento hasta adoptar el diseño a pantallas de gran tamaño.

Para aplicar este módulo, debemos tener en cuenta que todo lo que no sea esencial debe quedar fuera de nuestro diseño. Para hacerlo debemos pensar en lo que buscan nuestros consumidores, que es lo que no puede faltar en ningún caso y que es prescindible o superfluo. Teniendo en cuenta este enfoque de diseño, las dimensiones y capacidades de un móvil y la información que debemos incluir, podemos pasar al desarrollo que se debe incluir, podemos pasar al desarrollo de una página web.

Proceso del diseño:

Cuando la etapa de diseño para un dispositivo de tamaño reducido está lista, el siguiente paso es estirar la ventana hasta que el diseño hasta que empiece a deformarse o a verse mal. En este caso se definen los media queries para pantallas de mayor dimensión.

Ventajas del Mobile First:

La versión mobile necesita muchas adecuaciones en los elementos para que todo funcione en el espacio del celular. A veces algunos componentes se eliminan para que el sitio web quede perfecto en el ambiente. Agregar información en una pantalla más grande es relativamente más fácil a comparación del espacio de un dispositivo móvil, el cual no permite el exceso de información. Es necesario eliminar información de poca importancia para ofrecer una mejor experiencia al usuario.

Un sitio web hecho para dispositivos móviles se ajusta con más facilidad a la computadora. Además de que garantiza más visibilidad en los mecanismos de búsqueda, sitios web optimizados para dispositivos móviles suelen ofrecer una experiencia mejor para los usuarios y así aumenta la atención con la arquitectura de la información, usabilidad y accesibilidad.

Cuando se inicia la versión para desktop, el exceso de información no será un problema pues ha sido eliminado anticipadamente, solo hay que preocuparse por enriquecer las experiencias de usuario.

Retos al diseñar en Mobile Firts:

  • El tamaño de la pantalla reducida.
  • La conexión móvil limitada.
  • El comportamiento de los usuarios.
  • Las pantallas ‘touch’.

Ideas Clave:

  • El primer iphone de desarrollo en 2009 y despues en 2011 el mobile first.
  • No necesariamente se debe quitar el logotipo, se puede reducir su tamaño.
  • La informacion siempre estara cuando el usuario la quiere ver, no se satura de informacion la pantalla.
  • Estar conectado en internet se volvió parte del día a día y poco a poco se está dejando de usar la computadora para realizar las tareas cotidianas y cada vez se usa más los smartphones.
  • Lo primero se debe hacer al disenar en Mobile First es aclarar el contenido básico y separarlo del contenido sobrante o prescindible.
  • Pueden suceder casos en el que algún elemento no se incluya en la pantalla, si el espacio es muy pequeño, está la opción de crear un menú de barra con las opciones que no se puedan incluir en la pantalla.
  • Diseñar a partir de pantallas grandes es menos practico, requiere más esfuerzo reducir que ir aumentando el espacio.

Resumen:

Mobile First hace más sencilla la tarea de diseñar cualquier página web, de manera gradual el diseño se ira acomodando a los distintos tamaños de pantalla, adaptando el layout a pantallas más grandes. Este enfoque mejora la visibilidad en motores de búsqueda y facilita la adaptación a computadoras.

Las ventajas incluyen una mejor experiencia de usuario y una adaptación más sencilla a dispositivos de mayor tamaño. Sin embargo, presenta retos como el tamaño reducido de las pantallas y la conexión móvil limitada.

Día 30 de octubre: Modulo Grid

Notas de Clase:

¿Qué es Grid?:

Comúnmente se han utilizado varias estrategias de CSS para diseñar páginas web. Primero se usó table, luego float, position y display, pero todos estos métodos omiten funciones importantes (centrado vertical, por ejemplo). Grid es un sistema de diseño bidimensional basado en cuadriculas que, en comparación con cualquier sistema de diseño web del pasado, cambia por completo la forma en que diseñamos las interfaces de usuario.

Grid fue creado específicamente para resolver los problemas de diseño. A diferencia de flexbox cuyo flujo unidireccional tiene diferentes casos de uso, Grid permite definir un esquema de cuadricula.

display: grid;

Se debe definir un elemento contenedor como una cuadricula con esta propiedad. Todos los hijos directos del contenedor de la cuadricula se convierten automáticamente en elementos del grid.

Grid column:

Las lineas verticales de elementos grid son llamadas columnas.

.container {
display: grid; grid-template-columns: 1fr 2fr;
}

Grid rows:

Las lineas horizontales de elementos grid son llamados rows.

.container {
display: grid; grid-template-rows: 100px 200px;
}

column-gap

Los espacios entre las columnas se pueden ajustar con la propiedad column-gap.

.container {
display: grid; column-gap: 20px;
}

row-gap

Los espacios entre los rows se pueden ajustar con la propiedad row-gap.

.container {
display: grid; row-gap: 20px;
}

gap

La propiedad gap resume las propiedades anteriores, por ejemplo:

gap: 10px 20px;

fr

Es una unidad de medida de CSS Grid, es la abreviatura de “fracción” y se utiliza para especificar fracciones del espacio disponible en una cuadricula CSS. Es útil cuando se quiere distribuir el espacio restante después de colocar elementos en la cuadricula. Por ejemplo:

grid-template-column: 1fr 2fr;

Divide el espacio disponible en tres partes, asignando una fracción a la primera columna y dos fracciones a la segunda.

grid-template-areas:

El grid template-areas es una propiedad de CSS Grid que permite definir un esquema de cuadricula utilizando nombres de áreas personalizados en lugar de especificar las líneas de la cuadricula con grid-template-rows y grid-template-columns.

.container {
display: grid; grid-template-areas:
'header header'
'sidebar main'
'footer footer';
}
.header {
grid-area: header;
}
.sidebar {
grid-area: sidebar;
}
.main {
grid-area: main;
}
.footer {
grid-area: footer;
}

grid-column-start, gris-row-start y grid-column-end, gris-row-end

Determina la ubicación de un elemento de la cuadricula dentro de la cuadricula haciendo referencia a líneas de cuadricula específicas.

Ideas Clave:

  • La propiedad gap se puede usar de tres formas, columnas, renglones o los dos al mismo tiempo.
  • Hoy en día ya no hay problema con el soporte.
  • Al modificar los valores del gap, cambian el tamaño de los elementos que están dentro.
  • Si hay una altura o anchura determinada de los elementos, estos podrían salirse del contenedor dependiendo de la propiedad gap que se use.
  • Gracias a grid, ahora es más fácil maquetar una página web por su cuadricula.
  • Se puede utilizar display grid y display flexbox en un mismo documento, no hay uno que sea mejor que el otro y se complementan uno al otro.

Resumen:

CSS Grid es un sistema de diseño en cuadrícula que facilita la creación de interfaces web que a diferencia de métodos antiguos como table o float, grid permite un control más preciso, incluyendo el centrado vertical. Se define un contenedor con display y las líneas verticales se llaman columnas y las horizontales, filas.

Las propiedades como column-gap y row-gap ajustan los espacios entre columnas y filas. La unidad fr distribuye el espacio disponible en fracciones, usar grid simplifica el diseño de páginas web y puede usarse junto con Flexbox.

Día 4 de noviembre: Unidades de CSS:

Notas de Clase:

Unidades de CSS:

Las unidades absolutas, como px (pixeles), cm (centímetros) e in (pulgadas), representan tamaños fijos y no cambian independientemente del contexto en el que se utilicen, proporcionando una medida exacta que puede ser útil para diseños que requieren precisión milimétrica.

Unidades Relativas:

Las unidades relativas como % (porcentaje), em y rem, son flexibles y se adaptan en función de otros elementos. Por ejemplo % es relativo al tamaño de fuente del elemento padre, y rem es relativo al tamaño de fuente del elemento raíz (html).
Este tipo de unidades son ideales para crear diseños responsivos, ya que permiten que los elementos del diseño cambien en respuesta al tamaño de la pantalla.

Unidades de vista de ventana:

Las unidades de vista de ventana son herramientas que se ajustan dinámicamente al tamaño de la ventana grafica del navegador, las unidades son las siguientes:

  • vw y vh:
    El “vw” significa viewport width y representa un porcentaje del ancho de la ventana, donde 1vw es igual al 1% del ancho total de la ventana.
    El “vh” viewport height y representa un porcentaje de la altura de la ventana, donde 1vh es igual al 1% de la altura total de la ventana.
  • vmin y vmax:
    Estos valores toman el valor mínimo y máximo entre el ancho y la altura de la ventana, respectivamente, lo que permite una adaptabilidad aún mayor en situaciones donde la orientación de la pantalla puede variar.

Unidades Absolutas:

  • Centímetros
  • Milímetros
  • Pulgadas (1in = 96px = 2.54cm)
  • Pixeles (1px = 1/96 de 1 in)
  • Puntos (1pt = 1/72 de 1 in)
  • Picas (1pc = 12pt)

Unidades Dinámicas de Vista de Ventana:

Las unidades dinámicas ofrecen mayor precisión en el diseño responsivo porque toman en cuenta las variaciones de tamaño de la ventana grafica en dispositivos móviles. Estas unidades son las siguientes:

  • dvh
    La unidad dvh significa dynamic viewport height (altura del viewport dinámico) y representa el 1% de la altura de la ventana grafica dinámica, ajustándose en tiempo real a los cambios en la interfaz de usuario, como la aparición o desaparición de barras de navegación (búsqueda).
  • svh
    El svh significa small viewport height (altura del viewport pequeño) y corresponde al 1% de la altura de la ventana grafica más pequeña que el navegador ha registrado, proporcionando un valor estable que evita cambios bruscos en el layout.
  • lvh
    Large Viewport Height, se refiere al 1% de la altura de la ventana grafica más grande registrada, permitiendo disenar para el máximo espacio disponible sin que se vea afectado por cambios temporales en el viewport.

Al ser unidades relativamente nuevas, el soporte de navegador puede variar. Los navegadores modernos como las últimas versiones de Chrome, Firefox, Safari y Edge suelen ser los primeros en implementar estas nuevas características. Pero es posible que algunos navegadores más antiguos o menos utilizados no las soporten completamente aún.

Ideas Clave:

  • vw y vh no representan un porcentaje con respecto al elemento contenedor, si no ancho de la ventana.
  • Existen las unidades relativas “ex” y “ch” pero no se usan mucho.
  • Las "picas" no son una medida que se use mucho en la web, sino más bien en impresiones.
  • Al usar tipografías en ems, la tipografía de un elemento hijo puede pasar que se vea más grande que la del padre, por lo que no es muy recomendable utilizar ems mucho.
  • Al usar dvh y vh, en algunos navegadores para móviles, la barra de búsqueda y el link puede esconderse al realizar scroll, lo que puede generar pequeños cambios en el diseño según el viewport.
  • No hay recomendaciones de uso de unidades, ya que todas tienen sus diferentes reacciones en algunos navegadores.

Resumen:

Las unidades de CSS se dividen en absolutas y relativas. Las absolutas, como px, cm e in, tienen tamaños fijos, mientras que las relativas, como %, em y rem, se adaptan según otros elementos, siendo ideales para diseños responsivos.

Las unidades de vista de ventana (vw, vh, vmin, vmax) se ajustan al tamaño de la ventana del navegador, y las unidades dinámicas (dvh, svh, lvh) ofrecen mayor precisión en dispositivos móviles, cada tipo de unidad tiene sus usos específicos y puede comportarse de manera diferente según el navegador.

Día 6 de noviembre: La regla @supports en CSS

Notas de Clase:

@supports:

La regla @supports en CSS permite detectar si el navegador soporta una característica CSS específica y aplicar estilos condicionales basados en esta compatibilidad. Es útil para implementar funcionalidades modernas mientras se garantiza que los navegadores que no soportan estas características aun presenten un diseño funcional.

Mejora Progresiva:

Es una estrategia de diseño web que consiste en crear una versión básica de la funcionalidad para todos los navegadores, y luego agregar características mejoradas para navegadores que las soporten.

Degradación Elegante:

La degradación elegante se refiere a la estrategia de crear la versión más avanzada de funcionalidad y asegurarse de que se degrade de manera aceptable en navegadores más antiguos o menos capaces.

Sintaxis de @supports:

La estructura básica de esta propiedad es la siguiente:

@supports (propiedad: valor) {
elemento {
propiedad: valor; /*Estilos que se aplican si la característica es soportada*/
}
}

Condiciones:

La regla de @supports puede utilizar los siguientes operadores lógicos:

and:

Ambas condiciones deben ser verdaderas.

@supports (propiedad: valor) and (propiedad: valor) {
elemento {
propiedad: valor; /*Estilos que se aplican si la característica es soportada*/
}
}

or:

Al menos una de las condiciones debe ser verdadera.

@supports (propiedad: valor) or (propiedad: valor) {
elemento {
propiedad: valor; /*Estilos que se aplican al menos una característica es soportada*/
}
}

not:

La condición debe ser falsa.

@supports not (propiedad: valor) {
elemento {
propiedad: valor; /*Estilos que se aplican si la característica NO es soportada*/
}
}

Buenas Practicas:

  • Especificidad: Cuida que los estilos dentro de @supports cumplan con la especificidad y no afecten otros estilos.
  • Eficiencia: Evita condiciones muy complejas que puedan afectar el rendimiento.
  • Alternativas: Proporciona estilos alternativos para asegurar que el diseño sea funcional en navegadores que no soportan las características más modernas.

Ideas Clave:

  • El @supports funciona para navegadores que no tienen compatibilidad con algunas funciones sean detectadas.
  • La regla @supports tiene un buen soporte en los navegadores modernos, incluyendo las últimas versiones de Chrome, Firefox, Safari y Edge, excepto Internet Explorer.
  • No se tiene el control del navegador del usuario, si no de las funciones de la página.
  • CSS cada vez crea funciones que remplazan a Javascript, pero son funciones que algunos navegadores no soportan.
  • Ya no saldra CSS4, ya que ahora serán actualizaciones de parte de los navegadores.
  • Actualmente Safari está promoviendo una propiedad similar a display grid, llamado “mansory”.
  • Se pueden combinar operadores lógicos al utilizar @supports.

Resumen:

La regla @supports en CSS permite a los desarrolladores aplicar estilos condicionales según la compatibilidad de características en diferentes navegadores. Esto asegura una experiencia de usuario consistente al permitir el uso de nuevas características CSS, como display: grid, con estilos alternativos para navegadores que no las soportan.

Además, @supports puede combinar condiciones con operadores lógicos, ofreciendo flexibilidad para manejar diversos escenarios de compatibilidad.
